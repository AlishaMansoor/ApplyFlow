//MONGODB PIPELINE IF I WANT EVERY JOB OF A RECRUITER WITH ITS APPLICANT(APPLICATION) COUNT

two collections (think of them as SQL tables):
Jobs table:
_id          | title              | postedBy
-------------|--------------------|---------
job_1        | Frontend Dev       | user_123
job_2        | Backend Dev        | user_123
job_3        | Fullstack Dev      | user_123
Applications table:
_id          | jobId    | candidateId | status
-------------|----------|-------------|-------
app_1        | job_1    | cand_A      | Applied
app_2        | job_1    | cand_B      | Applied
app_3        | job_2    | cand_C      | Applied
app_4        | job_3    | cand_A      | Rejected
app_5        | job_3    | cand_D      | Applied

Pipeline runs step by step — each step transforms the data:

Step 1 — $match (WHERE in SQL):
sql-- SQL equivalent
SELECT * FROM jobs WHERE postedBy = 'user_123'
Result after $match:
_id   | title          | postedBy
------|----------------|----------
job_1 | Frontend Dev   | user_123
job_2 | Backend Dev    | user_123
job_3 | Fullstack Dev  | user_123
Only this recruiter's jobs pass through.

Step 2 — $lookup (LEFT JOIN in SQL):
js{ $lookup: {
    from: "applications",  // JOIN this collection
    localField: "_id",     // jobs._id
    foreignField: "jobId", // matches applications.jobId
    as: "applications"     // store result in this new field
}}
sql-- SQL equivalent
SELECT jobs.*, applications.*
FROM jobs
LEFT JOIN applications ON jobs._id = applications.jobId
WHERE jobs.postedBy = 'user_123'
Result after $lookup:
_id   | title         | applications (array)
------|---------------|------------------------------------------
job_1 | Frontend Dev  | [{app_1, cand_A}, {app_2, cand_B}]  -- 2 apps
job_2 | Backend Dev   | [{app_3, cand_C}]                   -- 1 app
job_3 | Fullstack Dev | [{app_4, cand_A}, {app_5, cand_D}]  -- 2 apps
Each job now has a full array of its applications attached.

Step 3 — $addFields (computed column in SQL):
js{ $addFields: {
    applicantsCount: { $size: "$applications" }
}}
sql-- SQL equivalent
SELECT *, COUNT(applications.jobId) as applicantsCount
FROM jobs LEFT JOIN applications...
GROUP BY jobs._id
Result after $addFields:
_id   | title         | applications                        | applicantsCount
------|---------------|-------------------------------------|----------------
job_1 | Frontend Dev  | [{app_1, cand_A}, {app_2, cand_B}]  | 2
job_2 | Backend Dev   | [{app_3, cand_C}]                   | 1
job_3 | Fullstack Dev | [{app_4, cand_A}, {app_5, cand_D}]  | 2
Just counted the array size and stored it as a new field.

Step 4 — $project (SELECT specific columns in SQL):
js{ $project: { applications: 0 }}
// 0 = exclude this field
// 1 = include this field
sql-- SQL equivalent
SELECT _id, title, postedBy, applicantsCount
-- exclude the full applications array
FROM ...
Result after $project:
_id   | title         | postedBy  | applicantsCount
------|---------------|-----------|----------------
job_1 | Frontend Dev  | user_123  | 2
job_2 | Backend Dev   | user_123  | 1
job_3 | Fullstack Dev | user_123  | 2
Full applications array removed — only count kept. Saves bandwidth!

Step 5 — $sort (ORDER BY in SQL):
js{ $sort: { createdAt: -1 }}
// -1 = descending (newest first)
// 1  = ascending (oldest first)
sql-- SQL equivalent
ORDER BY createdAt DESC

Final result sent to frontend:
json{
  "jobs": [
    { "_id": "job_3", "title": "Fullstack Dev", "applicantsCount": 2 },
    { "_id": "job_2", "title": "Backend Dev",   "applicantsCount": 1 },
    { "_id": "job_1", "title": "Frontend Dev",  "applicantsCount": 2 }
  ]
}

SQLMongoDB AggregationWHERE$matchLEFT JOIN$lookupSELECT col AS alias$addFieldsSELECT col1, col2$projectORDER BY$sortLIMIT$limitGROUP BY + COUNT$group
